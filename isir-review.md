## 1. 访问控制与安全服务（第 001-008 页）

### 1.1 ISO 7498-2 五类安全服务

ISO 在网络安全标准 ISO 7498-2 中定义了 5 类安全服务：

1. 身份认证服务：确认通信实体、用户或系统的真实身份。
2. 访问控制服务：限制主体对客体的访问，防止未授权使用资源。
3. 数据保密服务：防止数据被未授权泄露。
4. 数据完整性服务：防止数据被未授权修改、删除、插入或破坏。
5. 不可否认服务：防止通信参与者事后否认已经发生的行为。

### 1.2 访问控制三要素

访问控制包括三个基本要素：

- 主体（Subject）：主动发起访问请求的实体，例如用户、进程、程序、设备。
- 客体（Object）：被访问的资源，例如文件、数据库记录、设备、服务。
- 控制策略：规定主体对客体可以执行哪些操作，例如读、写、执行、删除。

可用一句话概括：访问控制解决“谁能对什么资源执行什么操作”的问题。

### 1.3 常见访问控制模型

- DAC（Discretionary Access Control，自主访问控制）：资源拥有者可自主决定谁能访问资源。优点是灵活，缺点是权限传播难控制。
- MAC（Mandatory Access Control，强制访问控制）：系统根据统一安全策略、主体安全级别和客体密级强制判断访问权限。常用于军事、涉密、高安全环境。
- RBAC（Role-Based Access Control，基于角色的访问控制）：用户通过角色获得权限，便于组织管理。常见于企业系统。
- 基于对象、基于任务、信息流模型等：分别从对象属性、任务流程、信息流动方向控制访问。

### 1.4 BLP 与 Biba 模型

BLP（Bell-LaPadula）模型主要保护机密性：

- 简单安全特性：不上读（no read up），主体不能读取高于自身安全级别的信息。
- 星特性：不下写（no write down），主体不能把信息写入低于自身安全级别的客体。
- 口诀：不上读，不下写。
- 局限：主要关注保密性，对完整性、安全管理、隐蔽通道等问题处理不足。

Biba 模型主要保护完整性：

- 不下读（no read down）：高完整性主体不读取低完整性数据，避免被污染。
- 不上写（no write up）：低完整性主体不能写入高完整性对象。
- 口诀：不下读，不上写。

| 模型 | 保护目标 | 读规则 | 写规则 | 典型考法 |
|---|---|---|---|---|
| BLP | 机密性 | 不上读 | 不下写 | 防泄密 |
| Biba | 完整性 | 不下读 | 不上写 | 防篡改/防污染 |

## 2. 网络安全基础、防火墙与 IDS（第 009-036 页）

### 2.1 OSI 与 TCP/IP 参考模型

OSI 七层从下到上：

1. 物理层 Physical
2. 数据链路层 Data Link
3. 网络层 Network
4. 传输层 Transport
5. 会话层 Session
6. 表示层 Presentation
7. 应用层 Application

TCP/IP 常用四层模型：

- 网络接口层 / Host-to-Network
- 网际层 / Internet / Network
- 传输层 / Transport
- 应用层 / Application

常考归属：

- IP 地址：网络层。
- MAC 地址：数据链路层。
- TCP/UDP 端口：传输层。
- HTTP、FTP、SMTP、DNS 等：应用层协议。

### 2.2 协议封装

数据从应用层向下传输时，每一层都会添加本层首部：

应用层数据 → TCP/UDP 段 → IP 包/分组 → 以太网帧 → 比特流。

解封装方向相反。示例中可见应用数据 Hello，经过 TCP 添加端口信息，IP 添加源/目的 IP，MAC 层添加源/目的 MAC。

### 2.3 IP 首部与 TCP 首部

IPv4 首部常见字段：

- 版本、首部长度、服务类型/区分服务、总长度。
- 标识、标志、片偏移。
- TTL、生存时间，用于限制包在网络中的生命周期。
- 协议字段，用来指明上层协议，如 TCP、UDP、ICMP。
- 首部校验和、源地址、目的地址、选项、数据。

TCP 首部常见字段：

- 源端口、目的端口。
- 序号、确认号。
- 首部长度、标志位、窗口大小、校验和、紧急指针、选项、数据。
- 常见标志位：SYN、ACK、FIN、RST、PSH、URG。

### 2.4 常用网络测试工具

| 工具 | 作用 |

| ping | 测试网络连通性，基于 ICMP |
| arp | 查看或解析 IP 与 MAC 地址映射 |

| traceroute / tracert | 追踪数据包经过的路径 |

| nslookup | 查询 DNS 解析结果 |

### 2.5 防火墙定义与应用

防火墙是在不同网络之间实施访问控制的系统，可以由硬件和软件组成。它通常位于内外网边界，用于保护内部网络，按规则允许合法通信、阻止非法通信，并可提供隔离、过滤、审计等能力。

防火墙应用场景：

- 内部网络与 Internet 的边界隔离。
- 对外服务可放在 DMZ 区，例如 Web、邮件、DNS。
- 基于规则控制内外网访问。

防火墙不是万能的：它不能替代主机安全、应用安全、IDS、日志审计和安全管理。

### 2.6 防火墙三类技术

| 技术 | 工作层次 | 效率 | 安全性 | 是否感知应用层状态 | 复习要点 |
|---|---|---|---|---|---|
| 包过滤 | 网络层/传输层 | 高 | 低 | 否 | 根据源/目的 IP、端口、协议、TCP 标志等判断 |
| 应用代理 | 应用层 | 低 | 高 | 是 | 代理客户端与服务器通信，可做内容检查、认证、日志 |
| 状态检测包过滤 | 网络层/传输层 | 较高 | 较高 | 是 | 维护连接状态表，结合上下文判断数据包 |

包过滤规则通常包含：

- 动作：允许或拒绝。
- 源 IP、目的 IP。
- 源端口、目的端口。
- 协议类型：TCP、UDP、ICMP 等。

规则设置原则：

- 最小权限原则：只允许必要通信。
- 默认拒绝：未明确允许的一律拒绝。
- HTTP 默认端口 80/TCP；若允许访问 Web 服务，应写明 TCP、目的端口 80、源/目的地址和允许动作。

### 2.8 为什么需要 IDS

需要 IDS 的原因：

- 入侵和攻击不断增多。
- 防火墙等传统安全产品不能完成全部安全防护任务。
- 防火墙偏边界访问控制，难以全面发现内部攻击、绕过防火墙的攻击、已发生入侵后的异常行为。

IDS（Intrusion Detection System，入侵检测系统）的核心作用：

- 收集主机、网络、日志等信息。
- 分析行为是否违反安全策略或符合攻击特征。
- 发现入侵、异常、违规行为。
- 产生报警、记录日志、辅助响应。

### 2.9 P2DR 动态安全模型

P2DR = Policy + Protection + Detection + Response。

- Policy：策略，是安全体系核心。
- Protection：防护，例如访问控制、防火墙、加固。
- Detection：检测，例如 IDS、日志分析、异常检测。
- Response：响应，例如告警、阻断、隔离、恢复、取证。

安全不是一次性产品，而是策略指导下的动态闭环。

### 2.10 IDS 结构、信息源与检测技术

CIDF 通用入侵检测框架包括：

- 事件产生器 Event Generator / E-box：采集事件。
- 事件分析器 Event Analyzer / A-box：分析判断。
- 事件数据库 Event Database / D-box：存储事件和日志。
- 响应单元 Response Unit / R-box：执行响应。

IDS 信息源：

- HIDS（Host-based IDS）：基于主机，分析系统日志、审计记录、文件变化、系统调用、用户行为等。
- NIDS（Network-based IDS）：基于网络，分析数据包、连接信息、协议流量等。

核心检测方法：

- 误用检测 / 特征检测：基于已知攻击特征或规则，准确率较高，但难以发现未知攻击。
- 异常检测：基于正常行为模型，能发现未知攻击，但误报率较高。

## 3. 恶意代码、信息安全属性与安全观（第 037-057 页）

### 3.1 病毒、蠕虫、后门

计算机病毒定义：编制者在计算机程序中插入的，破坏计算机功能或者毁坏数据、影响计算机使用，并能自我复制的一组计算机指令或者程序代码。

关键词：

- 人为编制。
- 插入或寄生于程序。
- 破坏功能或数据。
- 影响计算机使用。
- 自我复制。

蠕虫（第 039 页疑似）：通常不依附宿主文件，能利用系统漏洞或网络传播机制主动扩散。危害包括占用网络资源、造成拥塞、植入后门、传播恶意载荷等。

后门（第 040 页疑似）：系统中留下的非正常访问通道，可绕过正常认证，使攻击者再次进入系统。后门可能由攻击者植入，也可能来自恶意软件或不安全开发/配置。

第 038、041 页为恶意代码案例或相关特征页，文字较不清；可按病毒、蠕虫、木马、后门的传播方式、隐藏方式和危害方式对比复习。

### 3.2 信息安全属性

信息安全属性主要包括：

- 完整性 Integrity。
- 机密性 / 保密性 Confidentiality。
- 可用性 Availability。
- 不可抵赖性 Non-repudiation。
- 可控性 Controllability。

最核心的是 CIA 三要素：

| 属性 | 含义 | 典型破坏场景 | 常见保护技术 |
|---|---|---|---|
| 保密性 Confidentiality | 防止未授权访问和泄露 | 非授权用户读取敏感数据 | 加密、访问控制、身份认证、最小权限 |
| 完整性 Integrity | 防止未授权修改、删除、插入、伪造 | 数据被篡改、插入伪造信息 | 哈希、MAC、数字签名、权限控制 |
| 可用性 Availability | 授权用户需要时能正常访问和使用资源 | DoS/DDoS、系统宕机、链路中断 | 冗余、备份、容灾、抗 DDoS、监控 |

第 043-046 页分别通过图示说明：

- 可用性攻击：合法用户访问信息资源的通路被阻断。
- 保密性攻击：非授权用户非法访问信息资源。
- 完整性攻击：攻击者修改、插入、伪造信息，导致信息不可信。

### 3.3 进一步认识信息安全问题

第 053-056 页说明信息安全不是单纯技术问题，而是综合性、动态性、对抗性问题。

可从四个角度理解：

- 技术：系统复杂、软件漏洞、协议缺陷、攻击工具发展。
- 管理：制度不完善、权限管理混乱、应急响应不足。
- 人员：安全意识薄弱、口令泄露、社会工程学。
- 环境：开放网络、云计算、移动互联、供应链风险。

常见论述题答法：

- 绝对安全不存在，只能在成本、风险、效率之间平衡。
- 安全是持续过程，需要预防、检测、响应、恢复。
- 技术防护必须配合管理制度、人员培训和法律合规。

## 4. 密码学基础与信息隐藏（第 058-083 页）

### 4.1 密码学发展阶段

第 061 页给出三个阶段：

1. 1949 年以前：古典密码阶段。密码更多被视为技术，而不是科学。
2. 1949 年到 1976 年：Shannon 发表《Communication Theory of Secrecy Systems》，密码学建立理论基础。
3. 1976 年以后：Diffie 和 Hellman 发表《New Directions in Cryptography》，公开密钥密码体制出现。

高频时间点：

- 1949：Shannon。
- 1976：Diffie-Hellman。
- 1978：RSA。

### 4.2 古典密码与凯撒密码

古典密码包括：

- 代换密码：用其他字符替换原字符。
- 单表代换密码：如凯撒密码。
- 多表代换密码：如维吉尼亚密码（截图中疑似涉及）。
- 置换密码：不改变字符本身，只改变字符位置。

凯撒密码示例：

- 明文：`caesar was a great soldier`
- 密文：`fdhvdu zdv d juhdw vroglhu`
- 若 A、B、C... 对应 0、1、2...25，则加密公式：

```text
c = (p + k) mod 26
```

解密公式：

```text
p = (c - k) mod 26
```

练习页给出：

```text
c = (p - k) mod 26, k = 4
p = Beijing Information Science and Technology University
```

做题时注意：

- 只对字母做模 26 平移。
- 空格和大小写按题目要求处理。
- 凯撒密码密钥空间很小，可穷举破解。

### 4.3 密码系统基本概念

第 066-067 页为基本概念续页，文字部分较模糊，可按标准概念复习：

- 明文：加密前的信息。
- 密文：加密后的信息。
- 加密算法：将明文变换为密文。
- 解密算法：将密文还原为明文。
- 密钥：控制加密/解密过程的可变参数。
- 密码系统五元组：明文空间、密文空间、密钥空间、加密算法、解密算法。

单向陷门函数：

- 正向计算容易。
- 没有陷门信息时求逆困难。
- 掌握陷门信息时求逆容易。
- 公钥密码体制通常依赖这类思想。

### 4.4 对称密码、DES、IDEA、AES

对称密钥体制特点：

- 优点：速度快，所需资源少，适合大数据加密。
- 缺点：需要密钥交换；与未知实体初次通信困难；密钥管理复杂；可能依赖中心服务器。

DES：

- DES 已经过时，通常认为不再安全。
- 主要问题：有效密钥长度 56 位太短，容易被穷举。
- DES 常见参数：64 位分组、56 位有效密钥。

IDEA：

- 由 Xuejia Lai 和 James Massey 提出。
- 对称分组密码算法。
- 输入明文 64 位，密钥 128 位，输出密文 64 位。
- PGP 中曾实现 IDEA。
- 与 AES 区分：IDEA 分组 64 位，AES 分组 128 位。

AES：

- 1997 年 NIST 发起 AES 征集，用于取代 DES。
- AES 是分组密码，分组长度 128 bit。
- 支持密钥长度：128 bit、192 bit、256 bit。
- 1998 年公布 15 个候选算法。
- 1999 年公布 5 个候选算法：Mars、RC6、Rijndael、Serpent、Twofish。
- 2000 年 10 月 2 日，NIST 选择 Rijndael 作为 AES。

| 算法 | 类型 | 分组长度 | 密钥长度 | 状态 |
|---|---|---:|---:|---|
| DES | 对称分组 | 64 bit | 56 bit 有效密钥 | 已不安全 |
| IDEA | 对称分组 | 64 bit | 128 bit | PGP 曾使用 |
| AES / Rijndael | 对称分组 | 128 bit | 128/192/256 bit | 现代常用标准 |

### 4.5 RSA 与公钥密码

RSA：

- 由 Rivest、Shamir、Adleman 于 1978 年提出。
- 属于公开密钥密码算法。
- 安全基础是大整数因数分解困难：两个大素数相乘容易，但从乘积分解出两个大素数困难。

RSA 算法流程：

1. 选择两个互异的大素数 `p` 和 `q`。
2. 计算 `n = p × q`。
3. 计算 `φ(n) = (p - 1)(q - 1)`。
4. 选择整数 `e`，使 `e` 与 `φ(n)` 互素。
5. 公钥为 `(n, e)`。
6. 求 `d`，使 `e·d ≡ 1 mod φ(n)`。
7. 私钥为 `d`。
8. 加密：`y = x^e mod n`。
9. 解密：`x = y^d mod n`。

对称密码与公钥密码比较：

| 项目 | 对称密码 | 公钥密码 |
|---|---|---|
| 密钥 | 加密解密使用相同或容易互推的密钥 | 公钥公开，私钥保密 |
| 速度 | 快 | 慢 |
| 密钥分发 | 困难，需要安全信道 | 方便 |
| 适用 | 大量数据加密 | 密钥分发、数字签名、身份认证 |

混合加密：

1. 发送方生成会话密钥。
2. 用会话密钥对明文做对称加密。
3. 用接收方公钥加密会话密钥。
4. 接收方用私钥解密会话密钥。
5. 接收方用会话密钥解密数据。

优点：结合对称加密效率高和公钥加密密钥分发方便的优点。

### 4.6 信息隐藏

信息隐藏技术目录包括：

- 信息隐藏技术的发展历史。
- 信息隐藏的特点。
- 信息隐藏的方法。
- 信息隐藏的攻击。

信息隐藏与数据加密区别：

- 数据加密：隐藏内容，使攻击者看不懂密文，但通信存在本身可见。
- 信息隐藏：隐藏秘密信息或秘密通信的存在，使观察者不易发现有隐藏信息。

信息隐藏应用：

- 隐写通信。
- 数字水印。
- 版权保护。
- 完整性认证。
- 溯源取证。
- 隐蔽信道。

信息隐藏攻击分为：

- 隐藏技术：向载体对象中嵌入秘密信息。
- 隐藏攻击技术：检测、破解隐藏信息，或通过处理隐藏对象破坏隐藏信息、阻止秘密通信。

第 083 页为水印相关内容，疑似说明变换域方法相对空间域方法的优点：

- 嵌入水印能量可分布到空间域多个像素上，有利于不可见性。
- 可结合人类视觉系统 HVS 特性，提高隐蔽性。
- 可与国际数据压缩标准兼容，便于实现压缩域水印算法，如 DCT 水印。

## 5. 认证技术、数字签名、PKI 与法律法规（第 084-099 页）

### 5.1 数字签名

数字签名应满足：

- 收方能够证实发方的真实身份。
- 发方事后不能否认发送过的报文。
- 收方或非法者不能伪造、篡改报文。

数字签名提供：

- 身份认证。
- 报文完整性。
- 不可否认性。

数字签名不等于加密：

- 加密主要保护保密性。
- 签名主要证明“谁发的”和“有没有被改”。

### 5.2 数字签名原理

标准流程：

发送方：

1. 对报文 `M` 计算哈希摘要 `H(M)`。
2. 使用发送方私钥对摘要签名，生成签名 `S`。
3. 发送 `M + S`。

接收方：

1. 对收到的报文 `M` 计算 `H(M)`。
2. 使用发送方公钥验证签名，得到签名中的摘要。
3. 比较两个摘要是否一致。
4. 一致则说明签名有效；不一致说明身份不可信或报文被篡改。

直接数字签名：

- 技术上只涉及通信双方 A 和 B。
- 接收方需要知道发送方 A 的公开密钥。
- 发送方可用私钥对整个报文签名，但更好的方法是对哈希码签名，效率更高。

记忆：

- 加密：公钥加密，私钥解密。
- 签名：私钥签名，公钥验证。

### 5.3 认证技术分类

认证技术主要分为：

- 身份认证：认证用户身份或通信方身份，即确认“你是谁”。
- 报文认证：保证通信报文的完整性、来源真实性和不可抵赖性，即确认“消息是谁发的、有没有被改、能不能抵赖”。

身份认证三种基本方式：

| 类型 | 含义 | 示例 |
|---|---|---|
| 用户知道什么 | 知识因子 | 口令、PIN |
| 用户拥有什么 | 持有因子 | 磁卡、IC 卡、令牌、手机 |
| 用户是什么 | 生物特征因子 | 指纹、声音、DNA 图案、视网膜扫描、人脸 |

多因素认证 MFA：至少组合两类不同认证因子。

### 5.4 哈希函数

哈希函数特性：

- 输入可以是任意大小的数据块。
- 输出是定长摘要。
- 计算相对简单，易于软件或硬件实现。
- 单向性：给定摘要 `h`，找出 `x` 使 `H(x)=h` 在计算上不可行。
- 抗弱碰撞性：给定报文 `x`，找出不同报文 `y` 使 `H(y)=H(x)` 在计算上不可行。
- 抗强碰撞性：找出任意不同报文对 `x,y` 使 `H(x)=H(y)` 在计算上不可行。

SHA 与 MD4、MD5 比较页可辨认内容：

| 项目 | MD4 | SHA | MD5 |
|---|---:|---:|---:|
| Hash 值 | 128 bit | 160 bit | 128 bit |
| 分组块长度 | 512 bit | 512 bit | 512 bit |
| 基本字长 | 32 bit | 32 bit | 32 bit |
| 步数 | 48 | 80 | 64 |
| 步数结构 | 3×16 | 4×20 | 4×16 |

补充：MD4/MD5 已不适合安全用途，现代系统通常使用 SHA-2、SHA-3 等。

### 5.5 PKI 与 X.509 证书

PKI 的理论基础：

- 公钥密码理论。
- 公钥密码系统的核心是公钥和私钥。
- PKI 解决的问题：如何可信地绑定“身份”和“公钥”。

X.509 证书格式包含：

- 版本：1、2、3。
- 序列号。
- CA 的签名算法。
- 签发人名字 Issuer Name。
- 有效期 Validity Period。
- 主体名字 Subject Name。
- 主体公钥信息 Subject Public-key Information，包括算法、参数、密钥。
- 签发人唯一标识符。
- 主体唯一标识符。
- 扩展域 Extensions。
- 签名 Signature。

复习重点：

- X.509 证书 = 身份信息 + 公钥 + CA 签名。
- 证书里通常包含主体公钥，不包含主体私钥。
- CA 签名用于证明证书内容未被篡改，并确认身份与公钥的绑定关系。
- 验证证书时检查 CA 签名、有效期、主体名称、用途扩展、吊销状态。

复习重点：

## 7. 易混点专门对比

### BLP vs Biba

- BLP 保护机密性：防止信息从高密级流向低密级。
- Biba 保护完整性：防止低可信数据污染高可信对象。

### 防火墙 vs IDS

- 防火墙：主要做访问控制和阻断，属于防护。
- IDS：主要做检测和报警，属于检测。
- 有防火墙仍需要 IDS，因为防火墙无法发现所有内部攻击、绕过边界的攻击和事后异常。

### 误用检测 vs 异常检测

- 误用检测：基于已知攻击特征，误报较低，但难发现未知攻击。
- 异常检测：基于正常行为模型，可发现未知攻击，但误报较高。

### 加密 vs 数字签名

- 加密：保证保密性，典型流程是公钥加密、私钥解密。
- 数字签名：保证认证、完整性、不可否认，典型流程是私钥签名、公钥验证。

### 加密 vs 信息隐藏

- 加密隐藏内容，密文本身通常可见。
- 信息隐藏隐藏秘密信息的存在性，观察者不容易发现有秘密通信。

### 对称密码 vs 公钥密码

- 对称密码速度快，密钥分发难。
- 公钥密码速度慢，密钥分发方便。
- 实际系统常用混合加密：公钥保护会话密钥，对称密钥保护大量数据。

## 8. 可直接背的简答题模板

### 8.1 简述信息安全三要素

信息安全三要素是保密性、完整性和可用性。保密性指信息不被未授权主体访问或泄露；完整性指信息在存储、传输和处理过程中不被未授权修改、删除、插入或破坏；可用性指授权用户在需要时能够正常访问和使用信息资源。

### 8.2 简述 P2DR 模型

P2DR 是动态安全模型，由策略 Policy、防护 Protection、检测 Detection、响应 Response 四部分构成。策略是核心，用来指导防护措施部署；检测发现攻击或异常；响应负责处理和恢复。该模型强调信息安全是持续闭环过程。

### 8.3 为什么有防火墙还需要 IDS

防火墙主要在网络边界依据规则进行访问控制，不能完成全部安全防护任务，尤其难以发现内部攻击、绕过防火墙的攻击和已经发生的异常行为。IDS 通过收集和分析主机、网络、日志等信息发现入侵并报警，是防火墙的重要补充。

### 8.4 简述数字签名工作原理

发送方先对报文计算哈希摘要，再用自己的私钥对摘要进行签名，并将报文和签名一起发送。接收方收到后，对报文重新计算摘要，同时用发送方公钥验证签名并得到摘要，两者一致则说明报文未被篡改且来源可信。数字签名提供身份认证、完整性和不可否认性。

### 8.5 简述 RSA 的安全基础与基本公式

RSA 是典型公开密钥密码算法，安全性基于大整数因数分解困难。选择两个大素数 `p` 和 `q`，令 `n=pq`，`φ(n)=(p-1)(q-1)`，选择与 `φ(n)` 互素的 `e` 作为公钥指数，求 `d` 使 `ed≡1 mod φ(n)` 作为私钥指数。加密为 `y=x^e mod n`，解密为 `x=y^d mod n`。

### 8.6 简述 X.509 证书包含哪些内容

X.509 证书通常包含版本、序列号、签名算法、签发者、有效期、主体名称、主体公钥信息、唯一标识符、扩展域和 CA 签名。证书的作用是把主体身份与公钥可信绑定，证书中包含公钥而不是私钥。
